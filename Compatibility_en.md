# Compatibility (Interoperability / Integration)

PMP (php-plamodel-method) achieves “outer fixation × 10 grammar actions × unified logic”
and therefore coexists with existing frameworks, low-code tools, and API-based architectures
without conflict.  
Because the outer OS holds **no logic**, and fully separates UI, grammar, and core logic,
other technologies can be safely incorporated as “internal components.”

---

## Frameworks (Interoperability with Existing Frameworks)

Since the outer OS does not contain logic, it can integrate with frameworks such as  
Laravel, Symfony, CakePHP, and WordPress without any conflict.

- **parts** → can use framework views or components  
- **model** → translates framework features into the 10 grammar actions  
- **core** → encapsulates framework logic as a black box  

By treating existing frameworks as “internal components,”  
the outer OS gains both the flexibility of frameworks and the safety of an OS-level structure.

---

## Low-code / No-code (Integration with Low-code Tools)

UI development can be delegated to low-code tools (Bubble, Retool, AppSheet),  
while the grammar and logic layers remain fixed under the outer OS.

- Fast UI development through low-code  
- Unified outer grammar via the 10 actions  
- Single black-box logic in **core.php**  

The outer OS eliminates the typical low-code weakness of “scattered logic.”

---

## API / Microservices (Compatibility with API-based Architectures)

Because **core.php** is the only place where logic is implemented,  
API integration and microservice architectures become straightforward.

- **parts** → call external APIs  
- **model** → translate API responses into grammar actions  
- **core** → unify API logic inside a protected black box  

The outer OS keeps logic internal, ensuring safe boundaries with external services.

---

## Security (Structural Security Advantages)

The dependency-free design minimizes attack surface,  
and unified logic simplifies security management.

- Outer layer contains only parts/model  
- Inner layer contains only core (black box)  
- No external libraries → fewer vulnerabilities  

The outer OS structurally reduces attack vectors.

---

## AI-generated Code (Compatibility with AI-generated Components)

AI can generate **parts** and **model**,  
while humans maintain and protect **core**.

- AI generates UI and grammar layers  
- Humans safeguard the core logic  
- The 10 grammar actions prevent structural collapse  

In the AI era, the outer OS provides a structure that AI cannot break.

---

## Summary

The PMP outer OS is designed to safely incorporate existing technologies  
as internal components.

- No conflicts  
- Easy integration  
- Strong security  
- AI-ready architecture  

These characteristics give PMP exceptionally high compatibility  
and allow the outer OS to maximize its structural advantages.
