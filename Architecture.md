# Architecture Overview（外側OSの構造思想）

外側OS（Outer OS）は、アプリケーションの安全性・拡張性・一貫性を最大化するために  
「外側固定 × 文法10アクション × ロジック一本化」という構造原則に基づいて設計されています。

この文書は、PMP（php-plamodel-method）が採用する構造思想を体系的にまとめたものです。  
README は入口、SPEC_APPENDIX は仕様、Overview は構造図、Compatibility は外部との関係を示しますが、  
Architecture.md は **外側OSの“思想そのもの”** を記述します。

---

## 1. Outer Fixation（外側固定）

外側OSの中心思想は「外側固定」です。

外側固定とは、アプリケーションの構造を外側から固定し、  
UI・文法・ロジックの三層が常に一定の形を保つようにする設計思想です。

### 外側固定が必要な理由

- ロジックが UI に漏れると構造が崩壊する  
- 文法がロジックに混ざると安全性が落ちる  
- UI がロジックを直接触ると攻撃面が増える  
- 依存ライブラリが増えると寿命が短くなる  

外側固定はこれらの問題を構造的に防ぎます。

### 外側固定がもたらす効果

- **壊れない構造**  
- **安全性の最大化**  
- **拡張性の最大化**  
- **長期運用の容易さ**  
- **依存ゼロによる寿命の延長**

外側OSは「構造を守るための構造」です。

---

## 2. Grammar 10 Actions（文法10アクション）

外側OSは、アプリケーションの動作を 10 の最小単位に統一します。

- query  
- filter  
- sort  
- render  
- authenticate  
- save  
- update  
- delete  
- route  
- notify  

### 文法10アクションの役割

- model が「翻訳辞書」として機能するための文法  
- parts → model → core の流れを統一するための文法  
- 外側OS全体の動作を最小単位に分解するための文法  

### 文法10アクションがもたらす効果

- **構造の一貫性**  
- **処理の標準化**  
- **拡張時の衝突防止**  
- **ロジックの一本化を支える基盤**

文法10アクションは外側OSの“言語”です。

---

## 3. Black Box Logic（ロジック一本化）

core.php は唯一のロジック実装場所です。

### 黒箱（Black Box）の意味

- 外側から完全に隠される  
- parts や model から直接触れない  
- 内部構造を外側に漏らさない  
- ロジックを一本化することで安全性が最大化される

### 黒箱構造が必要な理由

- ロジックが UI に漏れると攻撃面が増える  
- ロジックが複数箇所に散らばると保守不能になる  
- 外側からロジックが見えると構造が壊れる  

### 黒箱構造の効果

- **安全性の最大化**  
- **拡張性の最大化**  
- **依存ゼロの実現**  
- **長期運用の容易さ**  

黒箱は外側OSの“心臓”です。

---

## 4. Three-Layer Separation（三層分離）

外側OSは次の三層で構成されます：

### 1. parts（UI）
- ユーザー操作を受け取る  
- 文法層に処理を委譲する  
- ロジックは書かない  
- 外側OSの入口  

### 2. model（文法 / 翻訳辞書）
- parts の要求を core が理解できる形に翻訳  
- 文法10アクションを使って処理を渡す  
- 外側OSの“言語層”  

### 3. core（ロジック / 黒箱）
- 唯一の処理実装場所  
- 外側から完全に隠される  
- 安全性と拡張性の中心  

### 三層分離の効果

- **UI・文法・ロジックの完全独立**  
- **構造の一貫性**  
- **壊れないアーキテクチャ**  
- **依存ゼロの実現**

### 補足：完全版における model / core の構造

この Architecture.md は外側OSの構造思想を説明しています。
完全版の構造では、model は次の二層に分かれます。

- **model.php** … 文法辞書（parts の要求を文法10アクションに翻訳する層）
- **model/** … 外側ロジック（利用者が記述する唯一の領域）

また、**core は文法（10アクション）を閉じて実行するためのロジックが記述される層**であり、
外側ロジックは model/ に記述します。

core と model.php は利用者が編集しない基盤層です。

---

## 5. Zero Dependency（依存ゼロ）

外側OSは外部ライブラリやフレームワークに依存しません。

### 依存ゼロの理由

- 外部依存は寿命を縮める  
- バージョンアップで構造が壊れる  
- セキュリティリスクが増える  
- 長期運用が難しくなる  

### 依存ゼロの効果

- **攻撃面が少ない**  
- **長期運用が容易**  
- **構造が壊れない**  
- **外側固定が成立する**

依存ゼロは外側OSの“寿命を伸ばす思想”です。

---

## 6. Minimal Example（最小構造）

外側OSの最小構造は次の 3 ファイルで成立します：

- parts/home.php  
- model/home.php  
- core/home.php  

この三層が揃うことで、  
外側固定・文法統一・黒箱構造が成立します。

---

## 7. Why Outer OS Exists（外側OSが存在する理由）

外側OSは「MVCの弱点を構造的に解決するため」に生まれました。

### MVCの弱点

- ロジックが UI に漏れやすい  
- Model が肥大化しやすい  
- Controller がロジックを持ちやすい  
- 依存が増えやすい  
- 長期運用が難しい  

### 外側OSの解決策

- UI はロジックを書かない（parts）  
- 文法は翻訳辞書として固定（model）  
- ロジックは黒箱に一本化（core）  
- 文法10アクションで構造を統一  
- 依存ゼロで寿命を伸ばす  

外側OSは「MVCの進化形」ではなく、  
**構造を守るための新しいアーキテクチャ**です。

---

## まとめ

外側OSは「構造を守るための構造」です。

- 外側固定  
- 文法10アクション  
- ロジック一本化  
- 三層分離  
- 依存ゼロ  
- MVCの弱点を構造的に解決  

これらの原則により、アプリケーションは安全で壊れず、  
どれだけ拡張しても構造が崩れない設計が実現します。

外側OSは、構造の安全性・拡張性・寿命を最大化するための  
**新しいアーキテクチャ思想**です。
