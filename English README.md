Plamodel-Style Programming Philosophy
The Original Doctrine of the Outer OS, Grammar Specification, and Black-Box Architecture
Author: S (Shou)

This philosophy, along with the Outer OS, grammar specification, and black-box architecture, was proposed, defined, and systematized by S (Shou).
This repository serves as the official documentation of the Outer Structure OS (Outer OS).

The philosophy is closely related to component-based programming, modular software architecture, grammar-based programming, and black-box architecture. It also aligns with modern AI-era programming paradigms and shares similarities with low-code / no-code approaches, while providing a more structured and scalable foundation.

1. Purpose of This Repository
This repository documents the original philosophy and technical structure of the Outer Structure OS (Outer OS), which is built on the following principles:

Fixing the grammar (10 Actions) on the outside

Black-boxing the internal logic (core)

A three-layer structure: public → model → core

Complete separation of grammar and logic

A programming structure optimized for the AI era

Just as natural languages can express infinite meanings with a small set of grammar rules,
these 10 Actions function as a minimal grammar that can generate unlimited application behavior.

2. Structure of the Outer OS
The Outer OS consists of the following three layers:

2.1 parts / flow (Outer Grammar)
Defines the application grammar externally. No logic is written here.

2.2 model (Receiver Layer)
Receives the outer grammar and passes it to the core.

2.3 core (Logic / Black Box)
Executes the actual processing. Completely separated from the outer grammar.

3. Black-Box Architecture
The core concept of the Outer OS:

Internal logic (core) is invisible from the outside

Complete separation of public and server

The outside contains only grammar

The inside contains only logic

The outside cannot modify the internal logic

This architecture ensures safety, rights protection, and market scalability.

4. Original Grammar (10-Action Specification)
The 10 basic actions that function as the grammar of the Outer OS:

query — Retrieve data

filter — Filter by conditions

sort — Sort data

render — Display or output

authenticate — Authenticate user or access

save — Create new data

update — Update existing data

delete — Delete data

route — Navigate or change screen

notify — Send notifications or events

These form the minimal grammar shared across all applications.
The core logic is executed only through these actions, ensuring a complete separation between grammar (outer structure) and logic (inner black box).

Plamodel-Style Programming Philosophy — Original Doctrine
7.1 Background of This Philosophy
Modern programming assumes that “writing code” is the default. However, in reality:

Beginners struggle with syntax

Experienced developers rewrite the same logic repeatedly

The market is flooded with rigid templates

AI is generating massive amounts of code

Language barriers still exist

This situation has reached a structural limit.

Therefore, I proposed a philosophy that transforms programming from
“writing code” to “assembling components.”

7.2 What Is the Plamodel-Style Approach?
The Plamodel-Style approach means:

Creating program logic as “parts” and running them through “configuration” and “combination.”

Writing code is not the goal — it is merely a means to assemble parts.

7.3 Problems This Philosophy Solves
Beginner Challenges
Syntax, errors, and environment setup

No visible results early on

Inefficiency for Experienced Developers
Rewriting the same logic

Switching mental models between languages

Market Limitations
Templates are “finished products” with low extensibility

Logic copyright is not protected

Contradiction in the AI Era
AI writes code

Humans still learn “writing-first” programming

The Plamodel-Style approach solves these structurally.

7.4 Core Structure of the Philosophy
Componentization
Logic is extracted as parts: single-purpose, reusable, and robust.

Configuration
Parts can change behavior through configuration files. Beginners only need to edit settings.

Combination
Complex features emerge by combining parts — just like assembling a plastic model.

Rights Structure
Logic copyright belongs to the author

Users only obtain usage rights

Viewing rights are separate

Modification requires permission

Market Structure
More parts → more value. Network effects similar to the App Store.

Language Independence
Works in PHP, JavaScript, Python, etc. The philosophy transcends technology.

Alignment with the AI Era
AI writes code. Humans handle understanding, judgment, and assembly.

7.5 Why Beginners Grow (Learning Model)
Works immediately

Only settings need to be edited

Hard to break

Touch → Change → Understand

Code is read only when necessary

This is the reverse of traditional learning.

7.6 Why Experienced Developers Grow (Extension Model)
Can add parts

Can modify parts (with permission)

Can develop across languages with the same philosophy

Can publish their own parts to the market

7.7 Market Formation (Economic Model)
A parts market naturally emerges:

Basic parts

Extended parts

Advanced parts

Viewing rights

Templates

Learning content

This creates a new programming economy.

7.8 Why This Philosophy Spreads Globally (Structural Advantage)
Beginners grow

Experienced developers benefit

Language-independent

Market expands automatically

Rights are protected

Optimized for the AI era

Structures engineering thinking

This is not just technology — it is a new industrial structure.

7.9 Formal Definition of the Outer OS
The Outer OS fixes the grammar externally and allows the internal logic to be freely replaced.

It consists of:

parts / flow (outer grammar)

model (receiver)

core (logic)

This achieves complete separation of grammar and logic.

7.10 Black-Box Architecture
A black box means:

Internal logic is invisible

public and server are fully separated

Outside = grammar

Inside = logic

Outside cannot modify the inside

This supports safety, rights protection, and market scalability.
